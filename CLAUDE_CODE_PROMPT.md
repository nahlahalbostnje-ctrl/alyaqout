## (4) بوابة الطالب (Student Portal)

> **ملاحظة:** هذه البوابة بحاجة لإعادة تصميم كاملة بنفس مستوى Admin/Parent (نفس نظام `const C`)، مع الميزات التالية مدمجة فيها من البداية.

### الصفحات الموجودة (تحتاج إعادة تصميم فقط):
`/student/dashboard` | `/student/courses` | `/student/live-classes` | `/student/exams` | `/student/homework` | `/student/report` | `/student/points` | `/student/league` | `/student/study-room` | `/student/courses/:courseId/content`

### ميزات جديدة تُضاف:

- **فيديوهات مراجعة نهائية:** قسم/تبويب منفصل ضمن `Student/CourseContentController` أو controller جديد، يعرض فيديوهات مخصصة للمراجعة (يمكن تمييزها بحقل `type` بجدول جديد). مع DRM-lite (تعطيل أزرار التحميل/الحفظ بالمتصفح)، مع زر "أسئلة" عائم فوق الفيديو يفتح نموذج أسئلة سريع.

- **زر "حاضنة المواهب":** صفحة جديدة، أول دخول → فورم/استبيان (الاسم، الموهبة، الصف، العمر، الهدف من الموهبة، الحلم). يُحفظ بجدول جديد مثلاً `talents`. ويظهر بعدها بروفايل الموهبة.

- **زر "صديق الدراسة الفردي":** مؤقت (Timer) للدراسة + حقل ملاحظات، مع خيار اختياري "وضع ذكي" يقترح فترات راحة/تنظيم بالاعتماد على مدة المؤقت فعلياً بدون الحاجة لـ AI (منطق بسيط — يمكن تطويره لاحقاً).

- **مكتبة الياقوت:** صفحة جديدة تعرض كتب/إهداءات/أسئلة سنوات سابقة/ملخصات. تُدار من الإدمن (CRUD) وتُعرض للطالب بحسب الدولة والصف.

- **مرشد الياقوت الطلابي:** نظام حجز موعد — شبيه بـ `emergency_requests` الموجود (أو امتداد له). الطالب يحجز موعد أو يكتب مشكلته، ويحصل على رد فوري (rule-based مبدئياً قابل للربط بـ `ChatbotService` لاحقاً).

- **التذكيرات + الكبسولة الزمنية الشهرية:** أول كل شهر، نموذج يكتب فيه الطالب هدف/رسالة لنفسه (تُحفظ + تُذكّر بتاريخ مستقبلي). استخدم `NotificationService` الموجود.

- **نظام التحديات:** تحدي فردي (الطالب يحدد هدفاً شخصياً ويتتبعه) + تحدي عائلي (يربط ولي الأمر بهدف مشترك مع الطالب: قراءة كتب، حل واجبات...). يمكن بناؤه فوق بنية شبيهة بـ `challenges` الموجودة، بجدول جديد `league_participants` / `leagues` و `challenge_participants`.

- **مرحلة التشجيع:** نظام نصائح تحفيزية تظهر بعد كل إنجاز (نقاط، اختبار، واجب) — رسائل قصيرة محفوظة بقاعدة بيانات `encouragement_messages` تُختار عشوائياً حسب نوع الإنجاز.

- **السبورة التفاعلية على Agora:** ⚠️ هذه تحتاج فحص فني أولاً — تأكد إن كانت ميزة الـ Whiteboard مدعومة فعلياً بنسخة Agora SDK المستخدمة بالمشروع (`AgoraService`) قبل البدء بأي تصميم واجهة لها.

- **رفع صور من الطالب للأسئلة/الواجبات:** امتداد لـ `homework_submissions` و `exam_submissions` لدعم نوع مرفق صورة، مع تحديد حد أقصى للحجم ونوع الملف.

- **إضافة صندوق تذكيرات/تنظيم مهام** على سطح لوحة التحكم الرئيسية (مكوّن صغير قابل لإعادة الاستخدام بكل البوابات).

- **مراجعة عامة لكل الواجهات:** ترتيب احترافي، تباعد موحّد، تناسق مع الـ Design Tokens، تجربة استخدام سهلة وسريعة.

## (6) بوابة ولي الأمر (Parent Portal)

- **تغيير اسم الميزة:** من "أولومبياد الآباء/الأبناء" إلى **"أولومبياد ولادي"** (تسمية واحدة موحّدة بكل الواجهات والـ routes المرتبطة، مثل `/parent/league`).

- **نظام تقسيط الأجهزة:** زر "طلب جهاز + اشتراك" بصفحة `/parent/billing` أو صفحة جديدة، يفتح فورم طلب (الجهاز، نوع التقسيط) يُحفظ كـ `lead` جديد أو جدول مخصص `device_requests`.

- **دورات لا منهجية:** قسم جديد بصفحة `/parent/academy` يعرض دورات (محادثة، ذكاء اصطناعي، إرشاد تربوي لولي الأمر) مصنّفة حسب الصف/المرحلة — استخدم نفس بنية `categories / courses` الموجودة مع تمييزها بحقل `is_extracurricular` أو `category` خاص.

- **تصميم بأسلوب تطبيقات البنوك:** تحديث الواجهة الرئيسية (`/parent/dashboard`) لتبدأ بترحيب ديناميكي ("صباح الخير محمد" / "مساء الخير محمد") حسب وقت اليوم واسم ولي الأمر.

- **إشعارات محاكاة متابعة لحظية:** استخدام `NotificationService` الموجود لإرسال رسائل دورية ("ابنك بحاجة يركز"، "ابنك اليوم أنجز قراءة") + ملخص يومي عبر واتساب (استخدام `whatsapp_number` / `whatsapp_default_message` بجدول `settings` الموجود).

- **مقارنات شهرية غير تراكمية:** بصفحة `/parent/academic-progress`، أضف رسم بياني يعرض التغيّر (delta) **بين كل شهر والشهر السابق له** بدل الرقم الكلي التراكمي فقط — استعلام يعتمد على `gamification_points` و `attendance_records` مجمّعة شهرياً.

- **إعادة ترتيب لوحة تحكم ولي الأمر (أولوية عالية):** رفع هذه العناصر لأعلى الواجهة الرئيسية:
  1. مؤشر التطور الأكاديمي
  2. المهام والتوصيات الذكية
  3. الأهداف العائلية
  - وتنزيل العناصر الثانوية (الإشعارات، عدد الأبناء، إلخ) للأسفل أو لشريط جانبي مصغّر.

## (8) الإدمن والسوبر إدمن (Admin / Super Admin)

- **مركز موافقات Super Admin → Admin:** نظام جديد شبيه ببنية `TeacherApprovalController/Admin` — يسمح للـ `Admin` برفع طلب (مثلاً: قرار، تعديل سعر، صلاحية) ويوافق/يرفض عليه الـ Super Admin — جدول جديد `approval_requests` (requested_by, type, payload JSON, status, reviewed_by).

- **إدارة المعلمين:** صفحة موسعة تعرض/تعدّل ساعات التدريس، الصلاحيات، الحوافز، والمواد لكل معلم — امتداد لـ `Admin/UserController` بحقول إضافية أو جدول `teacher_details`.

- **صلاحيات إدارية للطلاب (جديدة):** إضافة الإجراءات التالية بصفحة إدارة المستخدمين الحالية (`Admin/UserController`):
  - دمج حسابات (Merge Accounts)
  - تحويل الطالب لمعلم آخر (تعديل `course_id` / `teacher_id` المرتبط)
  - تغيير الباقة (ربط بـ `Admin/SubscriptionController`)
  - تعديل الاشتراك
  - سجل كامل لكل العمليات — Audit Log — جدول جديد `admin_action_logs` (admin_id, action, target_type, target_id, before, after, created_at)
  - نفس الصلاحيات أعلاه لإدارة أولياء الأمور.

- **قسم تحليلات إضافي:** صفحة جديدة تعرض أفضل معلم (بناء على تقييمات/أداء)، أكثر وقت دخول، معدل ساعات العمل، عدد ساعات فتح البرنامج — تحتاج تتبع جلسات الدخول (`آخر تسجيل دخول + مدة الجلسة`). يمكن إضافة جدول `login_sessions` أو استخدام بيانات JWT مع timestamp الموجود.
