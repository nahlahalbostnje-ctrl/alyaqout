## 1. الهدف الاستراتيجي للمشروع

التحول الكامل من نظام "الدورات المسجلة الثابتة" إلى منصة **فصول تفاعلية مباشرة** (Real-time Interactive Classes) تعتمد على:
- المتابعة المستمرة للطالب
- التلعيب (Gamification)
- المشاركة الفعالة بين الطالب والمعلم وولي الأمر

## 2. نظام تعدد الدول (Multi-Country Architecture)

المنصة تعمل على منصة واحدة تخدم عدة دول (الأردن، السعودية، الإمارات وغيرها)، مع عزل كامل للمحتوى بين الدول.

### مبدأ العمل
- الطالب يختار دولته عند التسجيل — يرى فقط محتوى دولته
- **Super Admin:** يرى جميع الدول ويديرها من لوحة تحكم واحدة
- **Admin الدولة:** يدير دولته فقط (محتوى، طلاب، معلمون، باقات، إشعارات)
- كل دولة لها: صفوف دراسية مستقلة، كورسات، باقات، أسعار، إعدادات

### الأدوار المحدّثة
| الدور | النطاق | الصلاحيات |
|-------|--------|-----------|
| `super_admin` | كل الدول | إدارة الدول، رؤية جميع البيانات، إنشاء admins |
| `admin` | دولة واحدة | إدارة محتوى دولته فقط |
| `teacher` | دولة واحدة | حصصه ومواده في دولته |
| `student` | دولة واحدة | محتوى دولته حسب اشتراكه |
| `parent` | دولة واحدة | أبناؤه في دولته |
| `supervisor` | دولة واحدة | طلابه المعيّنون في دولته |

## 2.1 المكدس التقني المعتمد (Tech Stack)

| الطبقة | التقنية | السبب |
|---|---|---|
| Backend | Laravel (PHP) | إدارة RBAC المعقد، APIs، ربط WaSenderAPI |
| قاعدة البيانات | MySQL / PostgreSQL | الهيكلية الهرمية للحسابات العائلية |
| Frontend | React.js + TypeScript | SPA سريع، دمج Agora SDK |
| Mobile | Flutter / React Native | WebView Wrapper + Native Push Notifications |
| Live Streaming | Agora.io | مدمج داخل التطبيق، زمن وصول < 400ms |
| Push Notifications | Firebase Cloud Messaging (FCM) | إشعارات الطلاب على الهاتف |
| Real-time Chat | Firebase Realtime Database | غرف الدردشة والدراسة اللحظية |
| WhatsApp | WaSenderAPI | إشعارات أولياء الأمور + OTP |
| Cloud & Hosting | Google Cloud | استضافة خوادم Agora والنسخ الاحتياطية |
| Video Protection | AWS CloudFront Signed URLs / VdoCipher | حماية الفيديوهات من التسريب |

## 4. هيكلية قاعدة البيانات (Database Schema)

### 4.1 الشجرة الهرمية للمحتوى

```
grades (الصفوف الدراسية)
  id, name
  └── categories (الفئات/المواد) [grade_id FK]
        id, name, parent_id, grade_id
        └── courses (الكورسات) [category_id FK]
              id, title, description, term_type, package_type, grade_id, category_id
              term_type ENUM: [foundation, first_semester, second_semester]
              └── units (الوحدات الدراسية) [course_id FK]
                    id, title, sort_order, course_id
                    └── lessons (الدروس) [unit_id FK]
                          id, title, sort_order, unit_id
                          └── videos (الفيديوهات والمرفقات) [lesson_id FK]
                                id, title, video_url (encrypted), duration, type, sort_order
```

### 4.2 جدول الصفوف الدراسية (grades)
```sql
id, name, created_at, updated_at
```

### 4.3 جدول الفئات (categories)
```sql
id, name, parent_id (null = رئيسي، رقم = فرعي), grade_id (FK), icon, sort_order
```

### 4.4 جدول الطلاب (students)
```sql
id, name, grade_id (FK - إلزامي), parent_id (FK), school, phone, region
```

### 4.5 جدول الباقات (packages)
```sql
id, name, description, price, duration_days, package_type ENUM[live_interactive, recorded_only],
discount_percentage
```

### 4.6 جدول الربط المحوري (package_subject)
```sql
package_id (FK), subject_id (FK)
```
> باقة مادة واحدة = صف واحد | باقة Bundle = عدة صفوف بنفس package_id

### 4.7 إعدادات الإشعارات والـ Chat Bot
```sql
settings: chatbot_api_key (encrypted), chatbot_system_prompt (text), whatsapp_number, whatsapp_default_message
```

## 5. الصفحة الرئيسية (Homepage)

### ما يجب إزالته
- أي قسم تحت مسمى "أحدث الدورات"
- روابط التعلم الذاتي المستقل

### الأزرار الرئيسية
1. **احجز الآن** — نموذج يحتوي: اسم الطالب، الصف، المدرسة، المواد المطلوبة، رقم الجوال، المنطقة السكنية
2. **اطلب حصتك المجانية الآن** — نفس النموذج (Lead Generation)

> **ملاحظة للنقاش مع صاحب المشروع:** هل يُضاف حقل خاص لنموذج الحصة المجانية لمعرفة احتياج الطالب بدقة لتسهيل الاستهداف التسويقي لاحقاً؟

### مسار التسجيل والدفع (Funnel)
```
تعبئة النموذج
   ↓
إرسال امتحان تحديد مستوى آلي
   ↓
توجيه لبوابة الدفع (باقات عائلية / فردية)
   ↓
جدولة الحصص + إشعارات المواعيد
```

### عناصر إضافية
- خريطة/قائمة المقرات مع زر "أرسل رسالة ليصلك أقرب مقر"
- **زر WhatsApp عائم (Floating Button)** في جميع الصفحات
- **Cookie Consent Banner** للموافقة على ملفات تعريف الارتباط

## 6. نظام التوثيق وتسجيل الدخول (Authentication)

### OTP عبر WhatsApp (بدل SMS)
1. المستخدم يدخل رقم هاتفه
2. Laravel يولد رمز OTP مؤقت (4-6 أرقام)
3. يُرسل عبر WaSenderAPI كرسالة WhatsApp
4. شاشة إدخال الرمز مع عداد زمني (60 ثانية) + زر "إعادة الإرسال"

## 10. نظام الاشتراكات والباقات (Packages & Subscriptions)

### 10.1 أنواع الباقات
| النوع | الوصف | صلاحيات الطالب |
|---|---|---|
| Live Interactive | صف تفاعلي مباشر | بث مباشر + Study Room + زر الطوارئ + تسجيلات الحصص |
| Recorded Only | فيديوهات مسجلة | مشاهدة VOD + PDF فقط (بدون بث مباشر أو دعم أكاديمي) |

### 10.2 هيكلية الباقات
- **باقة مادة واحدة**: ترتبط بـ subject_id واحد
- **باقة Bundle**: ترتبط بعدة subjects مع خصم تلقائي
- الخصم الآلي: إذا اشترى الطالب مادتين منفصلتين → يقترح النظام الباقة الشاملة بسعر أقل

### 10.3 آلية الوصول (Access Control)
```
student → subscription → package_type
  ├── live_interactive → يفتح: Agora Stream + Study Room + Emergency Button
  └── recorded_only   → يفتح: VOD Videos + PDFs فقط
```

> **سؤال لصاحب المشروع:**
> - هل سيكون الدفع عبر بوابات إلكترونية محلية (Jawwal Pay / PalPay)؟
> - هل يُسمح بالدفع اليدوي (حوالة بنكية + رفع إيصال للموافقة يدوياً)؟
> - هل تنتهي الباقة بتاريخ ثابت للجميع أم بمدة من تاريخ الشراء؟

## 12. نظام البث المباشر (Live Streaming - Agora.io)

### كيفية التكامل
- Agora SDK مدمج داخل React.js (لا روابط خارجية)
- واجهة مخصصة بالكامل بألوان ياقوت (White-label)
- Signed Tokens من Backend (Laravel) لمنع الدخول بدون اشتراك
- تسجيل سحابي تلقائي (Cloud Recording) لكل حصة

### مكونات الحصة المباشرة
```
┌─────────────────────────────────────────┐
│  فيديو المعلم (Agora Video)             │
├─────────────────────────────────────────┤
│  السبورة التفاعلية                       │
├──────────────────┬──────────────────────┤
│  دردشة جماعية   │  قائمة الطلاب        │
│  (Agora Chat)    │  (رفع اليد)          │
└──────────────────┴──────────────────────┘
```

> **سؤال لصاحب المشروع:** ما الحد الأقصى لعدد الطلاب في الحصة الواحدة؟ (حاسم لحساب تكاليف Agora)

## 17. ملاحظات تقنية هامة للمبرمج

1. **الكورسات الحالية**: تحويل حالتها من (دورات كاملة) إلى (مراجعات امتحانات) وربطها مجاناً بالباقات
2. **الكورسات اللامنهجية واللغات**: تبقى في E-commerce التقليدي مع تطبيق كود خصم آلي للمشتركين
3. **حماية الفيديو**: لا روابط مباشرة — استخدام Signed URLs (AWS CloudFront أو VdoCipher)
4. **فلترة الواجهات**: الطالب يرى فقط محتوى صفه (grade_id) تلقائياً عند تسجيل الدخول
5. **زر WhatsApp العائم**: رقم الهاتف والرسالة الافتراضية قابلان للتعديل من لوحة التحكم
6. **Cookie Consent**: تخزين الموافقة في Local Storage لعدم إزعاج المستخدم مرة ثانية
7. **المراجعات المجانية**: فتح تلقائي لكورسات مادة الطالب عند اقتراب الامتحانات
